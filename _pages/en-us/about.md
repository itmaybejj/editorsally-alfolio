[Editoria11y JS](https://editoria11y.princeton.edu) is an accessibility checker trusted by more than 70,000 websites to meet editors where they are:

* Tests run in real-time, offering *inline* corrections and advice as authors type.
* Tips use plain language that both correct and teach.
* Turnkey CMS plugins "just work;" they are installed in-site, not in-browser, so editors automatically see the results, even if they have never come to a training.

<h2 style="background: #d1d1f1;
padding: 1.5rem 1.5rem 1rem;
margin: 4.5rem -1.5rem 1.5rem;">The Editor's<strong>Ally</strong> community directs and supports ongoing feature development, documentation & translation.</h2>

### Completed initiatives
* Inline as-you-type checking in CKEditor, Gutenberg and TinyMCE
* A turnkey [Drupal Module](https://www.drupal.org/project/editoria11y/) & [WordPress Plugin](https://wordpress.org/plugins/editoria11y-accessibility-checker/), with reporting dashboards and alert dismissal systems

### In active development
* Site crawlers
* Developer test suites to complement the content test suite
* A custom test builder GUI
* Link checking

### Under consideration
* Pre-save checklists
* Flagged word checking
* CLI developer tools
* Bookmarklets and/or browser plugins
* Spellcheck
* Visual-diff based contrast detection for text over images
* Image of text recognition
* Context-aware alt text quality checking
